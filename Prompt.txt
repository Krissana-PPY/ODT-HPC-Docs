กลุ่มที่ 1 — วิเคราะห์และประเมินทรัพยากรก่อนส่งงาน
เหมาะสำหรับ user ที่ยังไม่รู้ว่าตัวเองควรขอ CPU/RAM/GPU เท่าไหร่
Prompt ตัวอย่าง:
ฉันต้องการรันโปรแกรม Python ที่ทำ [อธิบายงาน เช่น train neural network / วิเคราะห์ CSV ขนาด 10GB / จำลองข้อมูล 1 ล้านแถว] บนระบบ HPC ที่ใช้ Slurm โดยระบบมี Compute Node 2 เครื่อง แต่ละเครื่องมี CPU 64 cores, RAM 503 GB, GPU NVIDIA A40 และ partition ที่มีคือ cpu (max 7 วัน), gpu (max 3 วัน), short (max 1 ชม.) ช่วยแนะนำว่าควรขอทรัพยากรเท่าไหร่ และเขียน job script ให้ด้วย

กลุ่มที่ 2 — เขียน Job Script จาก code ที่มีอยู่แล้ว
เหมาะสำหรับ user ที่มี code พร้อมแต่ไม่รู้จะเขียน sbatch script ยังไง
Prompt ตัวอย่าง:
นี่คือ Python script ของฉัน: [วาง code ที่นี่] ช่วยเขียน Slurm job script (sbatch) ให้หน่อย โดยระบบมีข้อมูลดังนี้ — CPU: 64 cores/node, RAM: 503 GB/node, GPU: NVIDIA A40, partitions: cpu/gpu/short ช่วยเลือก partition ที่เหมาะสม ประเมิน CPU, RAM, และเวลาที่ควรขอ พร้อมอธิบายเหตุผลด้วย

กลุ่มที่ 3 — แก้ปัญหาเมื่องาน fail หรือมีพฤติกรรมผิดปกติ
เหมาะสำหรับ user ที่งานค้าง, fail, หรือไม่รู้ว่าเกิดอะไรขึ้น
Prompt ตัวอย่าง:
งาน Slurm ของฉันมีปัญหา สถานะแสดงว่า [PENDING / FAILED / TIMEOUT / OUT_OF_MEMORY] นี่คือ error log: [วาง log ที่นี่] และนี่คือ job script ที่ใช้: [วาง script ที่นี่] ช่วยวิเคราะห์ว่าเกิดจากอะไร และควรแก้ไขอย่างไร

กลุ่มที่ 4 — ปรับปรุง script ที่มีอยู่ให้ใช้ทรัพยากรดีขึ้น
เหมาะสำหรับ user ที่งานรันได้แล้วแต่อยากให้เร็วขึ้นหรือประหยัดทรัพยากร
Prompt ตัวอย่าง:
ฉันมี job script นี้: [วาง script] งานรันสำเร็จแต่ใช้เวลา [X ชั่วโมง] และ sacct บอกว่าใช้ RAM จริงแค่ [Y GB] จากที่ขอไว้ [Z GB] ช่วยแนะนำว่าควรปรับ --cpus-per-task, --mem, --time อย่างไรให้เหมาะสมขึ้น และมีวิธีทำให้งานเร็วขึ้นได้อีกไหม

กลุ่ม 5 — ตรวจสอบ script ก่อนส่งงาน
ช่วยตรวจสอบ job script นี้ก่อนที่ฉันจะ submit [วาง script] มี parameter อะไรที่ผิด, ขาดหาย, หรือไม่เหมาะสมกับระบบบ้าง

กลุ่ม 6 — วางแผนงานหลายขั้นตอน
ฉันต้องการทำงาน 3 ขั้นตอนตามลำดับ: 1) preprocess ข้อมูล 2) train model 3) evaluate ผล ช่วยออกแบบว่าควรแยกเป็นกี่ job และเขียน script แต่ละอันอย่างไร
